Qwen3.5-27B and Gemma4-31B on R9700

В данном отчете представлены базовые результаты для плотных моделей, которые полностью помещаются на одном графическом процессоре объемом 32 ГБ, а также приведены сравнения этих результатов с результатами тестирования на двух графических процессорах с разделением слоев, где это возможно.

Что такое плотные модели

Плотные модели активируют полную модель для каждого токена. В отличие от моделей MoE, здесь отсутствует разреженная маршрутизация экспертов, при которой в каждом прямом проходе участвует только подмножество экспертов. Это упрощает размещение плотных моделей: если квантованные веса и буферы выполнения помещаются в VRAM, вся модель может оставаться резидентной на одном графическом процессоре.

В этих тестах «помещение на один графический процессор» означает, что модель может работать с параметрами -sm none -mg 0 на одном графическом процессоре Radeon AI PRO R9700 объемом 32 ГБ без намеренной перегрузки компонентов модели в оперативную память процессора. Результаты для двух графических процессоров включены только для сравнения масштабируемости, а не потому, что модели требуют двух графических процессоров.

TL/DR:

2 GPU не ускоряют, а ощутимо замедляют работу плотных моделей. Это так для R9700, и как показывает поиск в сети — для всех остальных GPU.

Qwen3.5-35B-A3B and Gemma4-26B-A4B on R9700

В данном отчете приведены базовые показатели для двух MoE моделей, которые полностью помещаются на одном графическом процессоре объемом 31 ГБ, проведено сравнение размещения на одном и двух графических процессорах, а также проанализированы потери производительности при переносе экспертов MoE в оперативную память ЦП с помощью параметра ``--n-cpu-moe.**

Что такое модели M**oE

Модели MoE содержат множество экспертов, но для каждого токена активен лишь их поднабор. Это позволяет им иметь большое общее количество параметров, сохраняя при этом меньшую нагрузку на вычислительные ресурсы, чем у плотной модели того же размера.

В этих тестах обе модели MoE полностью помещаются на одном графическом процессоре при настройке UD-Q5_K_XL. Затем отчеты fit-moe принудительно переносят все большее количество экспертов в оперативную память ЦП с помощью параметра --n-cpu-moe 0,4,8,16,32,64, чтобы измерить затраты на выгрузку экспертов.
